# PRD — OrgTracker
**Product Requirements Document**
_Versión: 0.2_

---

## 1. Objetivo del producto

OrgTracker permite a un equipo de 1-2 personas gestionar las organizaciones clientes, asignarles los productos/sistemas que tienen contratados, y enviar notificaciones por email cuando hay una actualización o cambio de versión en alguno de esos productos.

El sistema no requiere base de datos tradicional — todo el almacenamiento es en archivos JSON en GitHub.

---

## 2. Usuarios y acceso

| Rol | Descripción | Acceso |
|-----|-------------|--------|
| Operador | Única persona con acceso | Todo: organizaciones, productos, anuncios |

- Autenticación: email + contraseña
- Las credenciales viven en un archivo de configuración (no base de datos)
- Solo 1-2 usuarios en total — no hay registro público
- Acceso vía URL directa con login

---

## 3. Entidades y campos

### Organización
| Campo | Tipo | Descripción |
|-------|------|-------------|
| `codigo` | string | Identificador único |
| `nombre` | string | Nombre de la organización |
| `indPepsico` | boolean | Indica si es cliente del grupo Pepsico/Matutano |
| `productos` | array de códigos | Productos asignados a esta org |
| `jiraOrgId` | string (uuid) | ID de la organización en Jira — clave de sincronización |

### Producto
| Campo | Tipo | Descripción |
|-------|------|-------------|
| `codigo` | string | Identificador único |
| `descripcion` | string | Nombre del producto |

**Productos conocidos (carga inicial):** Gesmobile, Rutaswin, Ahora, Portal Matutano, Gconector
La lista es **variable** — se pueden agregar, editar y eliminar productos desde la app.

### Anuncio (Notificación)
| Campo | Tipo | Descripción |
|-------|------|-------------|
| `codigo` | string | Identificador único |
| `titulo` | string | Título del anuncio |
| `mensaje` | string | Detalle del cambio o actualización |
| `adjunto` | string (url/base64) | Archivo adjunto opcional |
| `fecCreacion` | fecha | Fecha en que se registra |
| `fecEnvio` | fecha | Fecha programada de envío del email |
| `hora` | time | Hora de envío |
| `codProducto` | string | Producto al que aplica el anuncio |

### Usuario
| Campo | Tipo | Descripción |
|-------|------|-------------|
| `correo` | string | Email del operador |
| `clave` | string (hash) | Contraseña encriptada |

## 8. Integración con Jira

El sistema se conecta a la API de Jira para obtener los contactos (email) de cada organización al momento de enviar un anuncio. No se almacenan emails en el JSON local.

### Estrategia de integración
**Opción C — Token en archivo de configuración local (repo privado)**
El usuario operador carga manualmente el archivo `jira.config.json` con las credenciales. La app lo lee en memoria durante la sesión. El repo de GitHub es **privado**.

```json
{
  "domain": "aicom-team.atlassian.net",
  "email": "jira.adm@agsoft.es",
  "token": "****"
}
```

> El archivo `jira.config.json` está en `.gitignore` — nunca se sube al repositorio.

### Flujo de notificación con Jira
1. Operador carga `jira.config.json` al iniciar sesión
2. Operador crea un anuncio y selecciona el producto
3. App consulta Jira API usando el token en memoria → obtiene organizaciones con ese producto via `jiraOrgId`
4. App extrae emails de contacto de cada organización desde Jira
5. Se envía el email a esos contactos vía EmailJS
6. El anuncio queda marcado como enviado con fecha/hora real

### Endpoints Jira a usar
| Acción | Endpoint |
|--------|----------|
| Obtener organizaciones | `GET /rest/servicedeskapi/organization` |
| Obtener clientes de org | `GET /rest/servicedeskapi/organization/{id}/user` |

> ⚠️ **POR DEFINIR:** ¿El `codigo` de la Organización en OrgTracker coincide con el ID de organización en Jira, o necesitamos un campo de mapeo `jiraOrgId`?
No se necesita campo de mapeo , son codigos independientes
